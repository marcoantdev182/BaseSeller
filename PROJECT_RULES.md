# Project Rules

1. Toda animacao de scroll deve usar Lenis no scroll e GSAP no movimento.
2. Toda animacao deve ser suave, cinematografica e intencional.
3. Nenhum elemento importante deve aparecer ou sumir em menos de 0.6s.
4. Usar preferencialmente `power3.out` ou `expo.out`.
5. O site deve funcionar bem no mobile.
6. Se GSAP ou Lenis causarem conflito no mobile, corrigir sem quebrar o visual do desktop.
7. O projeto deve usar rotas bem organizadas.
8. Componentes devem ser reutilizaveis.
9. Separar regra de negocio, dados mockados e UI.
10. Nao criar codigo baguncado em um unico arquivo.
